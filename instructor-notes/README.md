## Bootcamp Philosophy

This bootcamp is built on three principles:

**1. Concept before syntax**
Students learn *why* FHE handles work the way they do before memorizing function signatures. The handle model section (Lesson 1.2) is the most important lesson in the entire bootcamp.

**2. Failure is the teacher**
Every anti-pattern lesson shows the broken code first. Students should see what goes wrong before seeing the fix. Resist the urge to skip to the correct answer.

**3. Build real things**
Every homework assignment produces a complete, working contract that could be deployed. No toy examples that wouldn't survive production.

## Grading Workflow

1. Students submit GitHub repo link via form
2. Instructor clones and runs `npm test`
3. Use rubric in each homework file for point allocation
4. Return feedback within 48 hours before next week starts
5. Students may resubmit once for partial credit recovery

---

## Recommended Cohort Size

- **Optimal:** 15–25 participants
- **Maximum:** 40 (beyond this, office hours become bottlenecks)
- **Minimum:** 8 (peer review groups need at least 3 people)
