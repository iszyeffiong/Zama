# FHE 101 — Plain Language Primer

No cryptography background required. This page explains FHEVM concepts in plain language.

## The Core Idea

Think of FHE like a **locked box that can do math**. You put a number in the box (encrypt it), hand the box to the blockchain, and the blockchain can add, subtract, and compare — all without ever opening the box. Only you (or someone you authorize) can open it to see the result.

## Three Rules to Always Remember

**1. Encrypted values are bound**
A ciphertext is tied to a specific contract address and user address. You cannot reuse an encrypted input from a different context or contract.

**2. Input proofs are mandatory**
Every encrypted input must come with a proof that it was created for *this* contract and *this* sender. This prevents replay attacks.

**3. Permissions are required to decrypt**
The contract must explicitly grant permission before anyone can read a result. Nothing decrypts automatically.

## The 3 Permission Functions

| Function | What it does |
|---|---|
| `FHE.allowThis(value)` | Lets the **contract** reuse this handle in future transactions |
| `FHE.allow(value, user)` | Lets a **specific user** decrypt this value off-chain |
| `FHE.allowTransient(value, contract)` | Grants **one-transaction** permission to another contract |

## Typical Flow

```
1. User encrypts a value off-chain
   (bound to contractAddress + userAddress)

2. User submits transaction:
   contract.store(encryptedValue, inputProof)

3. Contract verifies proof and stores the handle:
   euint64 val = FHE.fromExternal(encryptedValue, inputProof);
   FHE.allowThis(val);
   FHE.allow(val, msg.sender);

4. Contract performs FHE operations:
   euint64 result = FHE.add(val, FHE.asEuint64(10));
   FHE.allow(result, msg.sender);

5. User decrypts result off-chain using the SDK
```

## Public vs. User Decryption

**User decryption** (most common) — only the address that was granted `FHE.allow()` can read the result. Everything else stays hidden.

**Public decryption** — anyone can read the result after the contract calls `FHE.makePubliclyDecryptable()`. Only safe when the value is intentionally meant to be revealed (e.g., auction winner, vote total).

> ⚠️ `makePubliclyDecryptable` cannot be revoked. Use it only when you truly intend public disclosure.

## What You Cannot Do

- You **cannot** use an `ebool` in a Solidity `if()` statement — use `FHE.select()` instead
- You **cannot** divide by an encrypted value — the divisor must be plaintext
- You **cannot** read an encrypted value in a `view` function — view functions return handles, not plaintext
- You **cannot** reuse a ciphertext from a different contract or sender
